## ✨ 功能特性

- **智能配窝推荐** — 基于贪心算法 + Hall 条件优化，自动计算最优雄性分配方案
- **唯一依赖保护** — 自动识别并锁定只有一只雄性可覆盖的雌性精灵
- **位置图可视化** — 以 7×7 半格网格展示精灵窝位置，支持拖拽调整
- **距离约束线** — 自动绘制可交配雌雄之间的连线（曼哈顿 ≤ 2.5 && 切比雪夫 ≤ 2）
- **聚合坐标算法** — 当多个雄性高度重叠覆盖雌性时，使用预设模板优化摆放位置
- **半格精度** — 聚合模式下支持 0.5 格坐标精度，确保精灵窝间距严格 ≥ 1 格
- **配置导入/导出** — 可将当前配窝方案导出为 JSON 文件，方便存档分享
- **图片导出** — 支持将配窝方案和位置图分别导出为 PNG 图片
- **赛季异色支持** — 支持普通和异色（赛季限定）精灵分别计数
- **进化链联动** — 按蛋组或赛季搜索时自动连带展示进化链上的所有精灵

## 📖 使用指南

### 第一步：设置窝数量

在顶部输入框中设置窝的总数量（范围 `1-10`）。

> 雌性最多占 `窝数 − 1` 个位置，最少保留 1 个雄性窝。

### 第二步：选择精灵

| 操作 | 说明 |
|---|---|
| 点击 **「🌸 选择雌性」** | 打开雌性精灵选择弹窗 |
| 点击 **「♂️ 选择雄性」** | 打开雄性精灵选择弹窗 |
| 蛋组筛选 | 下拉选择特定蛋组 |
| 赛季异色筛选 | 下拉选择特定赛季 |
| 关键词搜索 | 输入精灵名称关键词 |
| 进化链联动 | 搜索结果自动包含整条进化链 |
| 异色数量 | 有赛季标签的精灵可分别设置普通/异色数量 |

> 已选精灵以标签形式展示在主面板，点击标签可直接编辑数量，点击 ✕ 可移除。

### 第三步：生成方案

点击 **「生成配窝方案」**，算法按以下流程执行：

1. **唯一依赖分析** — 找出只有一种雄性可交配的雌性 → 优先锁定配对
2. **贪心补充** — 选择能覆盖最多未覆盖雌性的雄性放入剩余空位
3. **空位兜底** — 若仍有空位，以最大化总兼容数为目标追加雄性
4. **Hall 条件优化** — 移除覆盖能力最弱的冗余雄性，始终保证 100% 覆盖
5. **结果展示** — 每只雄性显示覆盖详情、唯一依赖标记、未覆盖警告

### 第四步：生成位置图

点击 **「生成位置图」**，算法按以下流程执行：

1. **聚类检测** — 判断是否触发聚合坐标算法
2. **坐标安放** — 在 7×7 网格（14×14 细网格）上放置所有精灵窝
3. **连线绘制** — 展示可交配雌雄之间的兼容关系
4. **拖拽调整** — 可手动微调位置，自动半格吸附 + 防重叠

## 🧠 算法说明

### 配窝推荐

#### 唯一依赖分析

```
遍历所有雌性精灵：
  若该雌性只有 1 种雄性可交配，
  且该雄性库存仅剩 1 只
  → 标记为"唯一依赖"，强制锁定配对
```

#### 贪心补充

```
剩余空位 = 窝数 − 雌性数 − 已锁定雄性数
循环剩余空位次：
  在所有可用雄性中，
  选择"能覆盖最多尚未被覆盖的雌性"的那只
  → 加入方案，更新覆盖状态
```

#### Hall 条件优化

```
WHILE 当前雄性数 > 最大匹配数：
  移除覆盖雌性数最少的那只非锁定雄性
```

> 保证最终方案中每只雄性都"不可替代"。

### 位置图生成

#### 阶段一：聚类检测

| 轮次 | 触发条件 | 雄性数 | 兼容雌性阈值 |
|---|---|---|---|
| 第一轮（按蛋组分组） | 同有效蛋组的雄性恰好 2-3 只 | 2 | ≥ 4 |
| 第一轮（按蛋组分组） | 同有效蛋组的雄性恰好 2-3 只 | 3 | ≥ 4 |
| 第二轮（孤独高覆盖） | 无同组雄性、compatCount ≥ 4、无唯一依赖 | 2 | ≥ 7 |
| 第二轮（孤独高覆盖） | 同上 + 合并兼容雌性 ≤ 6 时尝试 3 只 | 3 | ≥ 4 |

> 优先单蛋组聚类（原始蛋组数少的雄性优先被选中）。

#### 阶段二：聚合安置

##### 有效蛋组计算

```
对雄性 M 的每个蛋组 g：
  1. 若精灵池中不存在任何含蛋组 g 的雌性 → 排除 g
  2. 若含蛋组 g 的所有雌性与 M 的蛋组完全一致 → 排除 g（冗余）

结果 ≥ 1 → 返回结果，否则返回原始蛋组
```

##### 两雄性模板

```
雄性坐标：  (0.5, 0)  (−0.5, 0)
雌性坐标：  (1, −1)  (1.5, 0)  (1, 1)  (0, 1)
           (−1, 1)  (−1.5, 0)  (−1, −1)  (0, −1)
```

##### 三雄性模板

```
雄性坐标：  (0.5, 0)  (−0.5, 0)  (0, −1)
雌性坐标：  (1.5, 0)  (1, −1)  (−1, −1)
           (−1.5, 0)  (0, −2)  (−0.5, 1)  (0.5, 1)
```

##### 确定性放置（模板外剩余精灵）

| 聚类类型 | 条件 | 雄性位置 | 雌性位置 |
|---|---|---|---|
| 2 雄性 + 有唯一依赖 | 与聚合雌性无共同蛋组 | `(0, −1)` | `(−1, −1)` `(0, −2)` |
| 2 雄性 + 有唯一依赖 | 与聚合雌性有共同蛋组 | `(−0.5, −1)` | `(−1.5, −1)` `(−0.5, −2)` |
| 2 雄性 + 无唯一依赖 | 剩余 ≤ 4 只 | `(−0.5, −1)` `(−1.5, −1)` | `(−1.5, 0)` `(−1, 1)` |
| 3 雄性 | 剩余 ≤ 3 只 | `(−0.5, 1)` `(0.5, 1)` | `(1.5, −1)` |

> 与聚合雌性有共同蛋组的雄性优先放入 `(−0.5, 1)`。

#### 阶段三：细网格求解（回退）

当确定性放置不适用时，回退到 14×14 细网格约束求解器：

- 雄性随机采样（优先中心 / 聚合雌性引力）
- 雌性 DFS+剪枝 满足所有距离约束
- 雄性和聚合雌性兼容 → 强制在曼哈顿 ≤ 4 细格内放置
- 多策略尝试（strict/loose × level 2/3）
- fallback 最小封锁

#### 阶段四：整数格求解（无聚合）

当没有触发聚合时，使用 7×7 整数网格求解器：

- 雄性随机采样 → 雌性 DFS 满足约束
- 无 3×3 封锁，不保证 Chebyshev ≥ 1

### 距离约束

#### 聚合模式（细网格）

| 关系 | minDist（细格） | maxDist（细格） | 含义 |
|---|---|---|---|
| 雄性 ↔ 唯一依赖雌性 | 2 | 2 | Chebyshev = 1 格 |
| 雄性（无唯一依赖）↔ 雌性 | 2 | 2（tight）/ 4（loose） | 优先 1 格，不行放宽到 2 格 |
| 雄性（有唯一依赖）↔ 非依赖雌性 | 2 | 4 | 不挤占唯一依赖空间 |
| 子雌性 ↔ 聚合雄性 | 2 | 8 | 曼哈顿 ≤ 4 格 |
| 所有精灵窝间距 | 3×3 封锁 | — | Chebyshev ≥ 1 格 |
| `canStillMeetTargets` 范围 | — | 4 | 曼哈顿 ≤ 2 格 |

#### 非聚合模式（整数网格）

| 关系 | minDist | maxDist | 含义 |
|---|---|---|---|
| 雄性 ↔ 唯一依赖雌性 | 1 | 2 | 曼哈顿 1-2 格 |
| 雄性（无唯一依赖 + 高覆盖）↔ 雌性 | 1 | 4 | 曼哈顿 ≤ 4 格 |
| 雄性（有唯一依赖）↔ 非依赖雌性 | 2 | 2 | 保持距离 |

### 合并与后处理

1. **子结果合并**：聚合坐标（真实坐标） + 子问题坐标 → 统一数组
2. **compactPlacement**：整体平移到原点（`minX`, `minY` → 0）
3. **centerPlacement**：整体居中到 7×7 网格

### 可视化规则

- **线条显示**：曼哈顿 ≤ 2.5 **且** 切比雪夫 ≤ 2 的兼容雌雄之间画绿色连线
- **颜色区分**：雌性粉色方块、雄性蓝色方块
- **锁定标记**：🔒 表示唯一依赖配对
- **异色标记**：⭐ 角标

## 🛠 配置导入/导出

### 导出

点击 **「导出配置」** 按钮，生成 JSON 文件下载。

### 导入

点击 **「导入配置」** 按钮，选择 `.json` 文件即可恢复之前的配窝方案。

### 配置格式

```json
{
  "nestCount": 10,
  "females": [
    { "id": 3011,"name": "恶魔狼", "count": 7,"shiny": true},
    { "id": 3029,"name": "奇丽草","count": 1,"shiny": true}
  ],
  "males": [
    {"id": 3011,"name": "恶魔狼", "count": 2, "shiny": true},
    {"id": 3029,"name": "奇丽草", "count": 1, "shiny": true}
  ]
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `nestCount` | `number` | 窝总数 |
| `females[].id` | `number` | 精灵 ID |
| `females[].name` | `string` | 精灵名称（可省略，导入时自动忽略） |
| `females[].count` | `number` | 数量 |
| `females[].shiny` | `boolean` | 是否异色 |
| `males` | `object[]` | 同 `females` 结构 |

## ⚠️ 注意事项

- **雌性上限**：雌性数量超过 `窝数 − 1` 时，输入框会自动限制
- **未覆盖警告**：位置图中以 ⚠️ 标记无法被任何雄性覆盖的雌性
- **拖拽避让**：拖拽精灵窝时自动进行 Chebyshev ≥ 1 避让，不会与其他精灵窝重叠
- **数据路径**：`pets.json` 和 `defines.json` 必须放在 `./data/` 目录下
- **HTTP 服务**：不支持 `file://` 协议直接打开，必须通过 HTTP 服务访问
- **浏览器兼容**：需要使用支持 ES6+ 的现代浏览器
