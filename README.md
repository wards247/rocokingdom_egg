git clone https://github.com/heartsarsonist/rockingdom_egg.git
cd rockingdom_egg

1. 设置精灵窝数量（上限 10，自动保留至少 1 个♂️雄性窝）。

5. 点击 **📍 生成位置图** 查看 7×7 放置方案，可拖拽调整精灵位置，并导出图片。

# 🧠 配窝算法说明
## 一、整体目标

在给定窝数（如10个）、若干雌性精灵（每只可指定数量）和若干雄性精灵库存（每只可指定数量）的前提下，自动推荐一组雄性来填满剩下的窝，使得：

- 所有雌性都能被至少一只雄性覆盖（可交配）。
- 雄性尽可能不闲置（每个雄性至少能配到一只雌性）。
- 优先保障唯一依赖关系（某雌性只能与唯一雄性交配）。

## 二、算法流程（`computeRecommendation` 函数）

### 1. 数据准备

- 遍历雌性数组，为每一只雌性个体生成唯一 ID（如 `f-0-0`）。
- 计算需要多少只雄性来填窝：`requiredMales = 窝数 - 雌性总数`。
- 建立雄性库存 `stockMap`，并计算每只雄性最多能配多少只雌性（`maleLimit`）。

## 2. 识别唯一依赖（Unique Dependency）

找出那些只能与唯一雄性交配的雌性，并且该雄性的库存只有 1 只。将它们优先绑定为 `reservedPairs`（预留配对），并标记该雄性为“已锁定”（`locked`）。

### 3. 贪心填充剩余雄性

- 计算尚未被覆盖的雌性 ID 集合 `uncoveredFemaleIds`。
- 在库存中循环选择新增的雄性（直到填满需要的数量），每次选择标准：
  1. 优先选能覆盖最多“尚未被覆盖”的雌性的雄性。
  2. 若平手，则选择能覆盖的雌性总数最多的雄性。
  3. 不能超过该雄性可配雌性的上限（`maleLimit`）。
- 如果还有空位但已无可扩展覆盖的雄性，则退而求其次，选覆盖总数最多的雄性填充。
- 最终得到 `allMaleSlots` 列表（包括预留的和额外选择的）。

### 4. 霍尔条件优化（Hall's Condition）

为了避免雄性闲置，算法会通过最大匹配检查是否所有雄性都有用：

- 对当前雄性列表进行二分图最大匹配（匈牙利算法），要求匹配数等于雄性数。
- 若匹配数不足，则逐步移除覆盖能力最弱的非锁定雄性，直到满足霍尔条件。

### 5. 计算最终方案

- 标记唯一依赖关系（`calcUniquePairs`），更新每个雄性的 `lockedForIds`。
- 统计未被任何雄性覆盖的雌性（`uncoveredFemales`），若存在则警告。
- 统计空窝（因优化而少放的雄性），并生成详细覆盖信息。

## 三、核心特点总结

| 特点 | 说明 |
|------|------|
| 唯一依赖优先 | 自动识别并优先保留那些只能与唯一雄性交配的雌雄配对，防止它们被拆散。 |
| 均衡覆盖 | 贪心选择时优先考虑尚未被覆盖的雌性，避免部分雌性被忽略。 |
| 杜绝闲置 | 通过霍尔条件剪枝，确保每只选出的雄性至少能配到一只雌性，不留无用雄性。 |
| 库存感知 | 严格依据用户输入的雄性库存数量进行推荐，不会超量使用同一雄性。 |
| 自动填充 | 根据窝数自动计算需要多少雄性，无需手动计算。 |
| 支持多只同名 | 允许雌性/雄性有多个个体，内部使用唯一 ID 区分。 |

这个方案在满足“全覆盖”的同时，尽量维持“唯一依赖”关系的稳定性，并避免浪费雄性资源，非常适合游戏中异色生蛋的策略需求。

###  智能♂️雄性推荐

- **唯一依赖优先**：若某只♀️雌性只能与唯一一种♂️雄性生蛋，则强制锁定该♂️雄性，避免遗漏配对。
- **贪婪覆盖**：剩余♂️雄性名额通过贪心算法分配，优先选择能覆盖最多未覆盖♀️雌性的♂️雄性。
- **防闲置机制**：♂️雄性总数不超过可配对♀️雌性种类数，杜绝“有窝无蛋”的浪费。
  
#  位置图算法说明
##  位置图逻辑

以♀️雌性精灵窝为起点，♂️雄性精灵窝为终点，♀️雌性只能通过上下左右移动（不能斜着移动）移动到♂️雄性精灵窝，所有♀️雌性行走速度一致，每只♀️雌性精灵有步数限制，每只♀️雌性在配窝方案中有几个可以与其交配的♂️雄性精灵，那么该♀️雌性就可移动几步（限定约束最多走两步），所有♀️雌性的位置需要满足不管她走向哪个可以与她交配的♂️雄性精灵窝，她都能在限定步数内达到目的地。
- 如果某个♀️雌性A（♀️雌性A限定步数不是1），她交配的♂️雄性A，这个♂️雄性A在配窝方案中可交配对象里有一个♀️雌性精灵B限定步数是1，那么将♀️雌性A到♂️雄性A的限定步数约束为2(不约束♀️雌性A到其他♂️雄性的限定步数)
- 如果♂️雄性A的可交配对象中没有限定步数为1的♀️雌性精灵，将♀️雌性A到♂️雄性A的限定距离约束为1，约束为1后无法找到合适位置，再将距离约束到2，
- 当给出的方案中♂️雄性A可交配的♀️雌性对象超过大于等于4,则♂️雄性A和其他♀️雌性之间的限定距离可放宽至4，但是♂️雄性A限定距离2的范围内必须至少存在3个可交配♀️雌性，如果没有则放宽条件2个可配♀️雌性即可
- 若♂️雄性A有唯一配对♀️雌性，则最后放置♂️雄性A及其唯一配对♀️雌性。若拥有唯一配对的♂️雄性精灵不止一个，则可配对精灵多的那个♂️雄性后放置精灵窝
- 每放置一只♀️雌性后，立即检查所有受影响的♂️雄性是否还有可能达到其“近距离目标”（3 或 2）。若某♂️雄性剩余未放置的♀️雌性中可交配的个体已不足以补足差额，则直接回溯，不再继续尝试当前分支。

##  曼哈顿距离布局

- 精灵窝分布在 7×7 网格上，配对有效的距离为曼哈顿距离 ≤ 2 步。
- 生成位置时优先保证“唯一依赖”的♀️雌性与锁定♂️雄性紧邻（距离=1）。
- 使用随机尝试 + 回溯搜索找到满足所有约束的布局，并自动居中、紧凑化。
